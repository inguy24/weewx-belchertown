Design Notes

Icon color fix: All icons on the colored left panel now use CSS inheritance — the .alert-icon-block sets color: #ffffff (dark bg) or color: #3d2800 (light bg, via data-bg-light="true"). No per-icon inline !important hacks. One rule governs all icons within a block. Dark backgrounds (red, orange, purple, green, steel-blue) use white icons. Light backgrounds (yellow, amber) use dark brown icons.

Flip-through rationale: Stacking N banners consumes N banner heights of screen real estate, which exceeds a full viewport on mobile when there are 3+ simultaneous alerts. The single-banner flip model takes 1 banner height regardless of alert count, while the pip row gives the operator an immediate severity overview.

Severity pips: Each pip is colored to match the alert's severity color — so the operator sees "purple + orange + yellow" before navigating and understands the severity distribution at a glance. Active pip is full opacity; inactive pips are 35% opacity.

Flip nav as button elements: The prev/next arrows are <button> elements, not <div onClick>. Each has a descriptive aria-label naming the target alert. Disabled at boundaries. Keyboard-reachable via Tab, activatable via Enter/Space.

Counter label: "1 of 3" text has aria-live="polite" so screen readers announce position changes when navigating. Not assertive — the operator chooses to navigate, so it's not an emergency announcement.

Single alert: When only one alert exists, the flip nav component is not rendered — no "1 of 1" counter, no arrows. The standard expand chevron appears instead, keeping the single-alert layout identical to v2.

Sorting: Alerts are ordered by severityLevel descending (4 → 1) per ADR-052. Navigation goes highest-first, so position 1 always shows the most severe alert.
