A4 — Card footprint system & grid-compatible sizing

This mockup visualizes the proposed card footprint system for the Clear Skies Now page. Each labeled box represents a real weather card sized to a fixed footprint (column-span × row-span) on a CSS grid. The grid engine itself is not built yet — these footprints are the sizing vocabulary being decided. Three device frames below show the same card set at 4-column (desktop ≥ 1024 px), 2-column (tablet ≥ 768 px), and 1-column (phone < 768 px) layouts, so you can judge proportions and responsive collapse. Cards use the real A1 design tokens: translucent glass surface, blur, radius, and spacing — layered over a dusk-sky gradient backdrop to make the translucency visible.

Footprint vocabulary

tile 1 col × 1 row — compact stat tile
wide 2 cols × 1 row — side-by-side pair
panel 3 cols × 1 row — wide content panel
full 4 cols × 1 row — full-width strip (hero / alerts)
tall +2 rows — cards that need vertical depth (dials, charts)
2×2 wide + tall — e.g. Wind Compass, Radar
4×1 full strip, single row — e.g. Active Alert, Highlights
Token values in use: --gap-grid: 1rem  ·  container-max: 80rem  ·  --card-radius: 0.875rem  ·  card surface ≈ 72% white (light) / 55% dark slate (dark) + blur(8px)
